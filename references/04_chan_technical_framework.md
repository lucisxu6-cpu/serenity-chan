# 04 Chan Technical Framework

## 1. Role Of Chan Analysis

缠论在本 Skill 中只解决一个问题：

```text
一个已经通过基本面筛选的对象，现在是不是高胜率介入/加仓/减仓位置？
```

它不能替代基本面。坏公司的技术买点仍然是坏公司的反弹；好公司的高位无结构追涨也不是高胜率。

---

## 2. Level System

长线研究至少看：

| 级别 | 用途 |
|---|---|
| 月线 | 产业趋势、大周期方向、估值中枢 |
| 周线 | 主升/主跌、估值消化、中期资金趋势 |
| 日线 | 主要操作级别，判断一买/二买/三买 |
| 30m/60m | 精细买点，辅助分批 |
| 5m/15m | 只用于执行，不用于长线判断 |

规则：

- 月线/周线决定能否长线；
- 日线决定是否可建仓；
- 30m/60m 决定分批位置；
- 小级别买点必须服从大级别趋势。

---

## 3. Core Concepts

| 概念 | 用法 |
|---|---|
| 笔 | 基础波段，用于力度比较 |
| 线段 | 多笔组成的结构 |
| 中枢 | 多段走势重叠区域，是成本和分歧区 |
| 背驰 | 同向离开中枢力度衰减 |
| 一买 | 下跌走势末端底背驰后的第一买点 |
| 二买 | 一买后反弹，再回调不创新低 |
| 三买 | 突破中枢后回踩不回中枢 |
| 一卖/二卖/三卖 | 与买点相反，用于减仓退出 |

---

## 4. Preferred Long-Term Buy Setups

### 4.1 二买：长线最舒服

条件：

```text
基本面未证伪
股价经历日线级别回调
下跌力度减弱或底背驰
第一波反弹出现
再次回调不破前低
成交/力度不再恶化
```

适合：高质量公司、产业逻辑强、短期因情绪/事件落地回调。

### 4.2 三买：强趋势确认

条件：

```text
基本面继续上修
价格突破日线/周线中枢
回踩不回中枢
再次向上确认
```

适合：强趋势股、不能等到深回调但又不想追突破当天。

### 4.3 一买：只做试错

一买便宜但难度高，只适合小仓试错。不能在基本面已证伪时用一买抄底。

## 5. Technical Health Assist: DMA / ATR

自动化完整缠论很难，AI/脚本可以用 DMA/ATR 做技术健康度预筛。

必须收集：

```text
latest close
20DMA / 50DMA / 100DMA / 200DMA
ATR20
5-day slope
20/50/100/200-day trend
relative performance vs index
```

### 5.1 Overheat Rules

| 状态 | 判断 |
|---|---|
| 价格高于 20DMA 0-5% | 健康贴线趋势 |
| 高于 20DMA 5-12% | 强势但略热 |
| 高于 20DMA 12-20% | 短期过热 |
| 高于 20DMA >20% | 逃逸，谨慎追高 |
| 高于 50DMA >30% | 中期过热 |
| 高于 100DMA >50% | 大级别重估 |
| 高于 200DMA >100% | 极端重估，除非基本面巨大上修 |

### 5.2 Pullback Rules

| 状态 | 判断 |
|---|---|
| 稳定基本面下回踩 20DMA/50DMA | 可能是健康回调 |
| 稳定基本面下跌破 50DMA 5-15% | 可观察二买，但需确认趋势未坏 |
| 跌破 100DMA 且财务预期下修 | 不是便宜，是趋势破坏 |
| 跌破 200DMA 且基本面恶化 | 剔除或重新建模 |

### 5.3 Escape Ratio

```text
EscapeRatio = 5-day price slope / 50DMA daily slope
```

- 0.8-1.2：健康平行；
- 1.2-1.8：加速但可解释；
- 1.8-2.5：明显发热；
- >2.5：FOMO 逃逸；
- <0：短期反转。

## 6. Chan + Fundamental Combined Rules

| 基本面 | 技术面 | 动作 |
|---|---|---|
| S/A 级 | 日线二买 | 核心候选建仓区 |
| S/A 级 | 周线三买 | 趋势确认，可中等仓 |
| S/A 级 | 高位逃逸 | 好公司但不追 |
| B 级 | 二买 | 小仓试错或观察 |
| C/D 级 | 任何买点 | 不做长线核心 |
| 基本面证伪 | 技术反弹 | 退出/不参与 |

---

## 7. Sell / De-Risk Rules

考虑减仓或重新评估：

- 周线顶背驰后反抽无力；
- 日线中枢下破后反抽不过；
- 三买失败变三卖；
- 基本面未继续上修但估值继续扩张；
- 财报证伪；
- 价格极度偏离 50/100/200DMA；
- 事件兑现后放量滞涨。

---

## 8. Output Requirement

技术部分必须写清：

```markdown
## 技术位置
- 月线：____
- 周线：____
- 日线：____
- 30m/60m：____
- 当前买点：一买 / 二买 / 三买 / 无买点 / 数据不足
- 如果无买点，等待条件：____
- 技术证伪：____
```

没有复权历史数据时，只能说“技术数据不足”，不能臆测买点。
