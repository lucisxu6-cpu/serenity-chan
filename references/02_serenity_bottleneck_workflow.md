# 02 Serenity Bottleneck Workflow

## 1. Core Principle

不要问“新闻厉不厉害”，要问：

```text
这个变化是否已经造成真实需求？
需求会流向哪条收入线？
哪家公司足够小、足够纯、足够靠近瓶颈，以至于这个需求能改写财报？
```

标准链条：

```text
news / policy / product launch
→ observable demand change
→ revenue/profit transmission
→ small-cap elasticity
→ market misclassification
→ validation path
→ falsification point
```

---

## 2. Theme Scan Workflow

### Step 1: Convert Narrative Into System Pressure

例子：

```text
国产大模型变强
→ 国产 AI 训练/推理需求提高
→ 昇腾/国产 GPU 集群扩张
→ 高速互联、HBM/DRAM、先进封装、液冷、电源受压
→ 找产能紧、认证慢、客户绑定强的上游公司
```

```text
人形机器人出货扩大
→ 关节/灵巧手/感知/力控用量增加
→ 丝杠、减速器、电机、编码器、力传感、3D 视觉受压
→ 找多家整机厂共同使用、外采比例高、量产难的部件
```

### Step 2: Build Value Chain Map

至少分为：

1. 终端需求/资金来源；
2. 下游客户/OEM/整机；
3. 系统/模块；
4. 核心器件/零部件；
5. 制造工艺/封装/测试；
6. 设备/材料/耗材；
7. 基础设施/电力/冷却/土地。

### Step 3: Rank Scarce Layers Before Companies

优先看：

- 低供应商数量；
- 长认证周期；
- 高工艺 know-how；
- 扩产慢；
- 客户更换成本高；
- 断供损失远大于采购成本；
- 产品在下游 BOM 占比小但关键性高；
- 市场仍按旧业务给公司分类。

### Step 4: Build Candidate Universe

广泛主题至少 20 家候选，覆盖：

- 明显龙头；
- 小市值高弹性供应商；
- 上游设备和材料；
- 封测/测试/计量；
- 基础设施；
- 私有公司和准上市公司；
- 热门但可能需要降级的公司。

如果工具、时间或数据源不足以完成 20+ 候选和广泛来源检查，必须把结论标记为 `initial pass`。不能用 3-5 个熟悉概念股伪装成深度扫描。

### Step 5: Layer-First Output Contract

主题扫描合格输出必须先出现：

```text
产业链地图
→ 瓶颈层级排序
→ 候选公司池
→ Top candidates
```

不合格模式：

- 先列公司，再反推产业链理由；
- 只列热门概念股，没有说明它们卡住哪一层；
- 把整机/OEM 热度当成上游瓶颈；
- 没有给出热门但降级的方向；
- 没有说明下一步用什么原始披露或财报字段验证。

### Step 6: Evidence Grading

| 等级 | 证据 |
|---|---|
| Strong | 公告、年报、招股书、SEC/HKEX/SSE/SZSE filing、合同、中标、监管文件、客户正式披露 |
| Medium | 公司 IR、电话会、产业会议、权威媒体、行业协会、专利/标准、上下游交叉验证 |
| Weak | 互动平台、普通媒体、KOL、社媒、渠道传闻 |
| Unverified | 无可追溯来源 |

Final top candidate 至少应有一个 Strong 或多个 Medium 交叉验证。只有 Weak 证据的公司不能排核心。

## 3. News-To-Financial-Statement Translation

必须把需求变成财务项目：

| 需求变化 | 财务传导 |
|---|---|
| 出货增加 | revenue units ↑ |
| 供不应求 | ASP / gross margin ↑ |
| 产能利用率提高 | fixed-cost absorption ↑ |
| 客户预付款/长协 | contract liability / backlog ↑ |
| 扩产 | capex / construction in progress ↑ |
| 认证成功 | revenue visibility ↑ |
| 低毛利集成 | revenue ↑ but margin risk |
| 贸易业务 | revenue ↑ but quality ↓ |

问：

```text
这条新闻最终会出现在哪一行报表？
收入、毛利率、现金流，还是只是市场情绪？
```

---

## 4. Small-Cap Elasticity

小市值弹性不是小市值本身，而是：

```text
alpha elasticity ≈ incremental demand impact / current revenue or market cap
```

高弹性条件：

- 公司收入基数小；
- 新需求足以改变收入/利润曲线；
- 业务纯度高；
- 供应链位置关键；
- 市场仍按旧标签估值；
- 1-4 个季度内可以验证。

大市值公司也可以是好公司，但若新业务对利润不够 material，不是 Serenity 式高弹性对象。

---

## 5. Market Misclassification

重点问：

```text
市场现在把它当成什么？
它是否正在变成另一个更高价值的类别？
这个再分类能不能被订单、毛利率、客户、产能验证？
```

例子：

- “普通化学品公司” → “先进制程 CMP 耗材平台”；
- “汽车零部件公司” → “机器人执行器核心供应商”；
- “连接器公司” → “AI 超节点高速互联瓶颈”；
- “老牌工业自动化” → “具身智能运动控制底座”。

---

## 6. Falsification Path

每个 thesis 必须有证伪点：

- 下游需求未兑现；
- 客户未进入量产；
- 第二供应商快速导入；
- 整机厂自研替代；
- 毛利率下降；
- 应收/存货恶化；
- 订单只是低毛利贸易；
- 估值已透支 3-5 年增长；
- 技术路线改变；
- 融资稀释严重；
- 周线主升结束且基本面未继续上修。

---

## 7. Red Flags

出现以下情况必须降级或保持观察：

- thesis 依赖单一客户传闻，但客户、订单和收入影响都没有原始披露；
- 公司声称稀缺，但毛利率、ASP 或利用率没有改善；
- 收入增长快于现金流，同时应收和存货明显恶化；
- 管理层频繁讲主题，但 segment disclosure 没有变化；
- 订单增长来自低毛利贸易、集成或一次性项目；
- 供应商数量快速增加，瓶颈层从紧缺变成竞争；
- 热门方向已经拥挤，但没有更高层级财务验证。
